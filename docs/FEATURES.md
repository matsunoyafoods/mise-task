## 認証・権限

### ロール
| ロール | 権限 |
|--------|------|
| 本部 | 全機能・全店舗閲覧・管理者PW必要 |
| 店長 | 自店舗の全機能 |
| スタッフ | 自店舗のタスク・仕込み・開閉店チェック |
| アルバイト | 自店舗のタスク・仕込み・開閉店チェック |

### 店舗切り替え（本部のみ）
- ヘッダーのセレクトから店舗を選択
- 切り替えには管理者パスワードが必要
- 「全店舗」表示に戻す際もパスワードが必要

## 発注管理フロー

```
在庫が「最低在庫」を下回る
        ↓
「発注候補」リストに自動追加
        ↓
店長が数量を確認して「発注する」
        ↓
発注確認モーダルで送信方法を確認
        ↓
LINEなら「LINEに自動送信」ボタン
        ↓
業者のLINEに発注内容が届く
        ↓
業者が納品 → 「検品」で在庫反映
```

---

## 業者管理

### 業者セルフ登録フロー
1. 本部が「QR / URL 発行」→ 業者名入力 → QR生成
2. QRをメール/LINE/FAXで業者に送付
3. 業者がスマホでスキャン → `supplier-reg.html` が開く
4. 3ステップで登録完了（会社情報・受注方法・連絡先）

### 対応受注方法
| 方法 | 動作 |
|------|------|
| 📧 メール | `mailto:` リンクが開く（手動送信） |
| 💬 LINE | Messaging API でPush自動送信 |
| 📸 Instagram | プロフィールが開く（手動DM） |
| ✕ X (Twitter) | プロフィールが開く（手動DM） |
| 📝 その他 | フリーテキストで表示 |

## localStorage 保存データ

全データはブラウザのlocalStorageに保存されます。
同じブラウザ・端末であればリロード後もデータが保持されます。

**データリセット方法:**
「スタッフ管理」画面（本部ログイン時）→「🗑 データ初期化」ボタン

## 技術的な特記事項

- **Reactバージョン**: 18.2.0（CDN経由、Babelでトランスパイル）
- **依存関係**: なし（package.jsonなし、npm不要）
- **ビルド不要**: index.htmlをそのままホスティングするだけで動く
- **スタイリング**: 全インラインスタイル（CSSファイルなし）
- **アイコン**: 絵文字のみ（画像ファイルなし）
